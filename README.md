# track-'n'-trace
universal osint tool - a framework for processing and displaying any data

- capable of working with any data desired as long as it contains either:
geospatial info; or,
time/vector info.

- is exceptionally light on resources,
  i.e low memory, low compute _and_ low disk churn

- can display on demand any and all data required at nodes, along vectors, with time and/or as flow/crossing/interaction/compare/etc.

This is an OSINT tool designed first and foremost for OSINT -not a global mapping tool for pretty maps or navigation pushed into service for OSINT, not an internet connectivity/telecoms tool pushed to carry/store OSINT type data and not a real time weather or traffic source twisted to cope with unsuited data types and flows. 

This is a custom designed first and foremost OSINT tool - built for data of multiple fields and types, with different time frame source streams, concurrently present by in different contexts to be compared as node mappings connected by vectors with desire to access detailed knowledge at any point or any join or vector of crossing or bottleneck etc.

- i) it displays as node/vector etc. 2d graph based on connections and the structure [not distance geo or presence of irrelevant concepts to data etc.]
- ii) it displays based on bottlenecks, flows, amounts and velocities etc. - i.e. w.r.t. time and mass
- iii) it displays globe geospatial visual mapped data for a graspable relation to real world as respects cities, major road/rail/flightpaths, hindrances such as oceans or deserts etc.


this means ot presents as a triple pane, data centred display _not_ a massive demand heavy graphics real time memory hungry/disk destroying super compute globe simulation with pretty pictures etc.

it is a usable and useful tool not a kludged second thought that doesn't really work.
